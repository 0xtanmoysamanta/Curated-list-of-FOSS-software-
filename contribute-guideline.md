## ğŸ“‹ Contribution Guidelines

### For Adding New Tools

#### âœ… What Makes a Good Addition?

A tool should be:
- **Free and Open Source** (FOSS) - The code must be freely available
- **Actively Maintained** - Regular updates within the last 12 months
- **Widely Used** - Has significant adoption in the developer community
- **Relevant** - Directly useful for software engineers' daily work
- **Documented** - Has clear documentation for users

#### âŒ What We Generally Don't Accept

- **Proprietary software** (even if free to use)
- **Abandoned projects** (no updates in 2+ years)
- **Niche tools** with very limited use cases
- **Forks without significant improvements** over the original
- **Tools still in early alpha** without proven stability

### Formatting Guidelines

When adding a tool, please follow this format:

```markdown
**Tool Name** emoji  
*Brief one-line description (under 80 characters)*
```

**Example:**
```markdown
**htop** ğŸ“Š  
*Interactive process viewer for Unix systems*
```

#### Emoji Guidelines
- Choose an emoji that represents the tool's function
- Keep it fun but relevant
- Don't repeat emojis too often in the same category

## ğŸ“ Style Guide

### Markdown Formatting

- Use **bold** for tool names: `**Docker**`
- Use *italics* for descriptions: `*Container platform*`
- Use consistent heading levels (we use `##` for categories)
- Add a blank line between entries for readability

### Writing Style

- **Be concise**: Aim for descriptions under 80 characters
- **Be accurate**: Ensure technical details are correct
- **Be neutral**: Avoid marketing language or hype
- **Be inclusive**: Use welcoming, accessible language

### Category Organization

When adding to existing categories:
- Maintain alphabetical order within subcategories when reasonable
- Place the most commonly used tools near the top if not alphabetical
- Keep related tools together

When proposing new categories:
- Ensure there are at least 3-4 tools that fit
- Use clear, descriptive category names
- Add an appropriate emoji for the section
